# ERP-006A PR-006A-10 — AI Engine Provider Settings

**Status:** **Implemented (local)** — settings UI + live Gemini Vision fallback for Courier PDF  
**Date:** 2026-07-14  
**Parent:** [ERP-006A-platform-consumer-model.md](./ERP-006A-platform-consumer-model.md)

## Goal

Allow system admins to configure Vision provider connection (API key + model) from the portal, and use those credentials for scanned PDF extraction on `/courier/pdf` without putting provider SDKs inside Courier.

## What this delivers

| Surface | Role |
|---------|------|
| `GET/PUT /api/ai-engine/settings` | Admin-only API; key never returned in clear text |
| `/ai-engine/settings` | Admin UI: enable, provider, model, API key |
| `data/ai-engine-settings.json` | Encrypted-at-rest key storage (gitignored) |
| `POST /api/courier/pdf/:id/reextract` | Re-run OCR → Vision on existing report |
| Upload path | OCR first; if `devices[]` empty → Gemini Vision via settings |

## Flow (scanned PDF)

```text
upload / reextract
  → OCR text layer
  → if devices empty AND settings.enabled + apiKey
  → rasterize PDF pages (pdfjs + canvas)
  → GeminiVisionAdapter (allowLive + image_payloads)
  → devices[] with extraction_source: ai_engine
```

## Supported providers / models

- **Gemini (live):** `gemini-2.0-flash`, `gemini-2.5-pro`, `gemini-1.5-pro`, `gemini-1.5-flash`
- OpenAI / Claude: selectable in UI — wiring deferred

## Explicit non-goals

- Enabling `AI_EXTRACTION_FEATURE_FLAG` production master switch
- Provider SDKs inside Courier modules
- Business Rules runtime (PR-006A-8)
- Billing / paid Gemini quota management

## Notes

- Free-tier Gemini may return HTTP 429 (quota) — UI surfaces `visionError`
- Production live Vision still awaits ERP-003 Pass for `vision_live_production`
